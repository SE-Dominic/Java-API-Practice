Java API Practice

This project is a simple Java-based API built to practice layered architecture and design patterns. It demonstrates how to structure a backend application using a Facade Pattern to simplify interactions with complex underlying systems.

Overview: 

The API is organized into multiple layers:

API Layer – Handles incoming requests and exposes endpoints.
Service Layer – Contains business logic and application rules.
Repository Layer – Manages data access and persistence logic.
Model Layer – Defines the core data structures used throughout the application.

A Facade Pattern is used to provide a simplified interface to the system, hiding the complexity of interactions between layers.
